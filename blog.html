  2. differences of localStorage, sessionStorage and cookies ?

    3. localStorage:

    allows us to store data on the user's computer without an expiration date. This means that the data will not delete even after the browser closes or computer turn off . It is useful for storing non-sensitive data such as user preferences, website settings, or user-generated content.

    sessionStorage:

    similar to localStorage, but the data is only available during the current browser session. This means that the data will be lost once the user closes the browser. It is useful for storing temporary data such as the contents of a shopping cart or a user's login information during a session

    Cookies:

    Cookies are small text files that are stored on the user's computer by the browser. They can be used to store data such as user preferences, login information, and tracking data. Cookies have an expiration date, so they can be used to store data that needs to be retained between browser sessions
